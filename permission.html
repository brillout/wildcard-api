Permission

Abort

We define permissions by using Abort.

// posts.telefunc.js
// Environment: Node.js

import { getContext, Abort } from 'telefunc'

export { deletePost }

async function deletePost(id) {
  // See the guide "Authentication" for an introduction about `getContext()`
  const context = getContext()

  // Only admins should be able to delete posts
  if( !context.user?.isAdmin ) {
    // The user is not an admin â€” we abort
    throw new Abort()
  }

  // ...
}

Telefunctions are public

We should never do something like this:

// query.telefunc.js
// Environment: Node.js

export async function executeQuery(query) {
  const result = await database.runSQL(query)
  return result
}

Telefunctions are public: anyone in the world can call our telefunctions.

// Environment: Anyone's browser dev console

// Anyone in the world can go to our website,
// open the browser's web dev console, and call our telefunctions.

const users = await executeQuery('SELECT login, password FROM users;')
users.forEach(({login, password}) => {
  // W00t I have all passwords ï½¡^â€¿^ï½¡
  console.log(login, password)
})

Instead we should use Abort:

// query.telefunc.js
// Environment: Node.js

import { getContext, Abort } from 'telefunc'

export async function executeQuery(query) {
  const context = getContext()

  // Only admins should be able to run arbitrary SQL queries.
  if( !context.user?.isAdmin ) {
    throw new Abort()
  }

  const result = await database.runSQL(query)
  return result
}

Abort or return someValue

There are situations where we don't want to Abort upon wrong permissions. For example:

// todo.telefunc.js
// Environment: Node.js

import { getContext } from 'telefunc'

export async function getTodoList() {
  const { user } = getContext()

  // The user is not logged in
  if( !user ) {
    // It is expected that the user may not be logged-in (for example when the user's session
    // expires). We `return { isNotLoggedIn: true }` instead of `throw new Abort()` so that
    // the frontend know what's going on and can redirect the user to the login page.
    return { isNotLoggedIn : true }
  }

  // ...
}

Note that Telefunc doesn't provide finer-grained exceptions (NotLoggedIn, InvalidFormInput, ...) because for fine-grain control we simply return JavaScript values instead of throwing exceptions.

We either throw new Abort() or return someValue.
